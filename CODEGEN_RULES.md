# Code Generation Rules (コード生成ルール)

このドキュメントは、Tauri を使ったアプリケーション開発において、コード生成・編集を行う際に必ず遵守すべきルールをまとめたものです。

---

## ✅ 1. 現在の状態は常にドキュメントで管理する

- アプリケーションの「現在の実装状況（実装済み/未実装）」は、常にドキュメントとして分かりやすく管理します。
- 何か変更を行う前に、ドキュメントを確認し、現在の状態を把握します。
- 変更を行った場合は、必ずドキュメントに反映します。

## ✅ 2. 手動のコード変更後の確認

- あなたが手動でコードを変更したと通知した場合（"確認してください" など）、以下を行います：
  1. 変更内容を確認する
  2. ドキュメントに変更点を反映する

## ✅ 3. 型エラー発生時の調査手順

- 型エラーなどが発生した場合は、以下を必ず行います：
  1. 依存ライブラリ（特に Tauri や Svelte）のバージョンを確認する
  2. 当該バージョンの公式ドキュメントや仕様を調べ、エラー原因を特定する
  3. 仕様違いによるエラーの場合でも、勝手に修正せずに必ずあなたの指示を仰ぐ

---

✨ **このドキュメントは必ず遵守します。**

- 以降、コードによる変更・生成を行う際は、このルールに従い、必要に応じてドキュメントを更新します。
