# Tauri Video Clip App Architecture

## 目的

Tauriで動画クリップアプリを作成し、大容量動画（数GB）でもクラッシュせずに以下を実現する。

-   動画再生
-   シーク
-   波形表示
-   低メモリ使用

------------------------------------------------------------------------

# 1. 基本方針

フロントエンドに動画ファイルを直接読み込ませない。

理由：

フロントで

-   Uint8Array
-   Blob
-   loadSourceFromPath

などを使うと

    video file
    ↓
    JS memory

となり、大容量動画でメモリ不足クラッシュが起きる。

------------------------------------------------------------------------

# 2. 動画再生アーキテクチャ

動画はRust側でストリーム配信し、フロントはURLとして読み込む。

    video file
    ↓
    Rust (Tauri custom protocol)
    ↓
    stream://video/{id}
    ↓
    video element

フロント側

``` javascript
video.src = "stream://video/123"
```

------------------------------------------------------------------------

# 3. ストリーミングの仕組み

video element は HTTP Range request を使用する。

例

    Range: bytes=2000000-

Rust側

    file.seek(2000000)
    ↓
    必要な部分だけ読む
    ↓
    chunk送信

動画サイズ4GBでもメモリ使用量は数十KB程度。

------------------------------------------------------------------------

# 4. 動画ストリーム処理

Rustは動画全体を保持しない。

    disk
    ↓
    small buffer
    ↓
    WebView

例

    64KB buffer

動画サイズに関係なくメモリ使用量は一定。

------------------------------------------------------------------------

# 5. 波形表示

WaveSurferは通常フロント側で音声デコードを行うが、長時間動画ではメモリ消費が大きい。\
そのためRust側でピーク生成を行う。

------------------------------------------------------------------------

# 6. ピーク生成方式

ピーク生成もストリーム処理。

    video file
    ↓
    demux
    ↓
    audio decode
    ↓
    sample chunk
    ↓
    peak計算
    ↓
    破棄
    ↓
    次のchunk

重要

    全データを処理する
    ≠
    全データをメモリ保持する

メモリ使用量

    数KB〜数十KB

------------------------------------------------------------------------

# 7. ピーク計算方法

一定サンプルごとに最大振幅を計算。

例

    window = 1024 samples

処理

    decode samples
    ↓
    abs(sample)
    ↓
    max value
    ↓
    peaks.push()

------------------------------------------------------------------------

# 8. ピーク生成タイミング

動画ロード時にバックグラウンド処理。

    動画読み込み
    ↓
    video 再生開始
    ↓
    Rust background task
    ↓
    audio decode
    ↓
    peak生成
    ↓
    wavesurfer更新

結果

    再生 → 即開始
    波形 → 数秒後

------------------------------------------------------------------------

# 9. wavesurferへのデータ供給

``` javascript
wavesurfer.load(url, peaks)
```

------------------------------------------------------------------------

# 10. 動画ファイルの読み込み回数

動画は2回読み込まれる。

### 動画再生

    disk → video decoder

### 波形生成

    disk → audio decoder → peak計算

OSファイルキャッシュにより負荷は小さい。

------------------------------------------------------------------------

# 11. キャッシュ

同じ動画を何度も開かない場合、ピークキャッシュは必須ではない。

必要なら

    video.mp4
    video.mp4.peaks

のように保存できる。

------------------------------------------------------------------------

# 12. 推奨アーキテクチャ

    video file
    │
    ├─ Rust custom protocol
    │   ↓
    │   stream://video/{id}
    │   ↓
    │   video element (再生)
    │
    └─ Rust background task
        ↓
        audio decode
        ↓
        peak生成
        ↓
        wavesurfer (波形)

------------------------------------------------------------------------

# 13. メリット

-   数GB動画でも動作
-   JSメモリ使用量が小さい
-   即再生
-   シーク可能
-   波形表示可能

------------------------------------------------------------------------
